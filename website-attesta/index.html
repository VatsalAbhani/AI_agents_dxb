§01 / THE PROBLEM

Agents are non-deterministic software
taking real-world actions.

You can't reproduce a failure.

A run depends on model outputs, tool responses, timing and randomness. When an agent misbehaves, that exact run is gone forever — debugging is guesswork.

You can't prove what happened.

Ordinary logs are mutable. Whoever holds the database can edit history — so a log is an account, not evidence. Auditors, customers and counterparties have to trust you.

Logging isn't control.

Observability tools tell you what happened after the message reached a customer. The dangerous moment is before.

Aviation solved this sixty years ago. It's called a flight recorder.

§02 / FOUR PRIMITIVES

One core, four guarantees.

  1. 01

    RECORD

    Every non-deterministic boundary — LLM calls, tool calls, time, randomness — is captured into an append-only, hash-chained ledger. Each entry commits to the previous one; the run is sealed with a Merkle root and signature.

  2. 02

    VERIFY

    Recompute the chain offline — no API, no server. Change one comma in one entry and every fingerprint after it stops matching. Independent public verification arrives with asymmetric signing and anchoring (see §04).

  3. 03

    REPLAY

    Re-run an agent against its recorded inputs. Replay never makes a live call — no rogue messages, no double payments. Divergence is detected and reported, honestly.

  4. 04

    PROVE

    A gateway checks policy and requires human approval before actions execute — and the whole run becomes a sealed, client-ready audit report after.

§04 / STRAIGHT ANSWERS

What we won't claim.

Can you replay an LLM bit-for-bit?
No — and nobody honestly can. Replay serves recorded outputs and cryptographically reports reproduced vs diverged. Verifiable divergence detection, not a magic trick.
Is the v0 seal asymmetric?
Not yet. v0 signs with HMAC — tamper-evident, but the key-holder could re-sign. Ed25519 signing and public transparency-log anchoring are the next core milestone; until they ship, we say "tamper-evident", never "independently verifiable".
Isn't logging enough?
Logging is testimony after the fact. Attesta adds control before it — policy checks and human approval gate the action itself. Block, approve, prove.
